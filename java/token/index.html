关于JWT的安全问题

JWT token 的关键过程点就是密钥, 加入这个密钥泄露了, 那么基本就可以伪造token了

这一点可以在用户登陆时 利用随机数生成token的TOKEN_SECRET , 并保存到缓存, key是登陆账户 , 客户端

访问接口是 : header要带登陆账号 和 token ,服务端拿到登录账号，到缓存去捞相应的SecretKey ，然后再进行token校验。可以防伪造token了（这个方案在一定程度上能防止伪造

，但是不能防止token泄露被劫持）
